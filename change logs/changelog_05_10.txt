routes/favourites_routes.py
----------------------------
ORIGINAL CHANGES


- (_load_team_matches): NS branch now skips team-specific query and falls
  through to the league-level date query.

- (get_home_screen): Replaced silent fallback logic with an explicit match feed
  mode system. Accepts a new "mode" query parameter ("all" or "my_teams").
  "All Matches" mode loads the general date/league feed. "My Teams" mode loads
  favourite team/league specific data with no fallback — empty sections show
  a clear "no matches for your teams" message instead of silently swapping
  to unrelated data.
    FROM: Team-specific query with silent fallback to league data when empty
    TO:   if use_my_teams:
              load team/league specific matches, no fallback
          else:
              load general date/league feed

- (_cache_home_payload_key): Added match_feed_mode to the cache key so "all"
  and "my_teams" modes are cached separately. Previously both modes returned
  the same cached response.
    FROM: (user_id, league_code, team_ids, player_ids, ...)
    TO:   (user_id, league_code, match_feed_mode, team_ids, player_ids, ...)

routes/match_routes.py
-----------------------
ORIGINAL CHANGES
- (get_fixtures): Replaced season + from/to range query with a simple

    FROM: params = {"league": league_id, "season": CURRENT_SEASON + 1,
                    "from": ..., "to": ...}

================================================================================
FREE PLAN API LIMITATIONS DISCOVERED
================================================================================

The following API-Football free plan restrictions were identified during this
session and have been worked around in the code:

- league + date requires season — cannot filter fixtures by league with a
  date parameter alone
- team + date requires season — cannot filter fixtures by team with a date
  parameter alone
- Date queries are restricted to today +1 day only — querying dates further
  than 2 days ahead returns a plan error
